## 1. Introduction to Data Analytics

### What is Data Analytics?

Data Analytics (DA) is the science of analyzing raw data to extract meaningful insights, identify patterns, and make data-driven decisions. In today's business landscape, data analytics has become the cornerstone of strategic decision-making, operational efficiency, and competitive advantage.

### Why Data Analytics Matters for MBA Professionals

For MBA students and future business leaders, data analytics is no longer optional—it's essential. The ability to:
- **Interpret business metrics** and KPIs effectively
- **Identify trends** in market behavior and consumer patterns
- **Optimize operations** through data-driven insights
- **Make strategic decisions** backed by empirical evidence
- **Communicate insights** to stakeholders clearly

## 2. Origin Story & Historical Evolution

### The Genesis of Data Analytics

**1960s-1970s:** Birth of Business Intelligence
- Early database systems and relational databases (Edgar F. Codd)
- Manual statistical analysis for business decisions
- IBM's development of mainframe computers for data processing

**1980s-1990s:** The Spreadsheet Revolution
- Introduction of VisiCalc (1979) and Microsoft Excel (1985)
- Data warehousing concepts emerge
- OLAP (Online Analytical Processing) systems

**2000s:** The Big Data Era Begins
- Internet explosion generates massive data volumes
- Google's MapReduce and Hadoop revolutionize data processing
- Business Intelligence tools become mainstream (Tableau founded 2003)

**2010s:** Advanced Analytics & AI Integration
- Cloud computing enables scalable analytics
- Predictive analytics and machine learning go mainstream
- Self-service BI democratizes data analysis

### Current Technology Landscape (2025)

**Core Technologies:**
1. **Business Intelligence Platforms**
   - Power BI, Tableau, Looker, Qlik
   - Real-time dashboards and interactive visualizations
   - Self-service analytics for non-technical users

2. **Big Data Processing**
   - Apache Spark, Snowflake, Databricks
   - Cloud data warehouses (AWS Redshift, Google BigQuery, Azure Synapse)
   - Stream processing for real-time analytics

3. **AI & Machine Learning**
   - AutoML platforms (Google Vertex AI, Azure ML, AWS SageMaker)
   - Predictive analytics and forecasting
   - Natural Language Processing for text analytics

4. **Modern Data Stack**
   - dbt (data build tool) for transformation

## 5. Technical Demo & Workshop: Hands-On Data Analytics

### Workshop Overview: Building an Analytics Pipeline with AI Tools

**Objective:** Create a complete data analytics workflow from raw data to actionable insights using modern AI-powered tools.

**Tools in Action:**
- Google Search Filters (Data Collection)
- Gemini LLM (Data Transformation)
- Claude AI (Dashboard Creation)
- Critical Thinking Framework (Insight Derivation)

### 5.1 Phase 1: Data Scraping with Google Search Filters

**Scenario:** Extracting business data from Google for competitive analysis

**Techniques:**
1. **Advanced Google Search Operators**
   ```
   site:linkedin.com "data analyst" "bangalore" filetype:csv
   inurl:data OR inurl:statistics site:gov OR site:edu
   "quarterly results" filetype:xlsx 2024..2025
   ```

2. **Scraping CSV Data**
   - Use browser extensions (Data Scraper, Web Scraper)
   - Identify structured data tables on web pages
   - Export to CSV format for processing

3. **Ethical Considerations**
   - Respect robots.txt and terms of service
   - Use public data sources
   - Comply with data privacy regulations (GDPR, CCPA)

**Hands-On Exercise:**
- Search for "MBA salary data India CSV"
- Extract structured salary information
- Save as `salary_data.csv`

### 5.2 Phase 2: CSV to Google Sheets Using Gemini LLM

**Why Google Sheets?**
- Collaborative and cloud-based
- Easy to share with stakeholders
- Integrates with visualization tools

**Using Gemini AI for Data Transformation:**

**Step-by-Step Process:**

1. **Upload CSV to Gemini**
   ```
   Prompt: "I have a CSV file with columns: Name, Role, Salary, Experience. 
   Please help me clean this data and create a Google Sheets formula 
   to calculate average salary by experience level."
   ```

2. **Data Cleaning with AI**
   - Remove duplicates
   - Handle missing values
   - Standardize formats (dates, currencies)
   - Detect and flag anomalies

3. **Generate Google Sheets Formulas**
   ```
   Gemini Output:
   =AVERAGEIF(D:D, ">5", C:C)  // Avg salary for 5+ years experience
   =COUNTIF(B:B, "Data Analyst")  // Count of data analysts
   =VLOOKUP(A2, SalaryTable, 3, FALSE)  // Lookup salary by name
   ```

4. **Create Pivot Tables**
   - Gemini can generate pivot table configurations
   - Automatic summarization by categories
   - Dynamic filtering and grouping

**Real-World Application:**
Imagine you're analyzing our seminar registration data (44 students):
- Gemini can calculate average ratings by year (First vs Second)
- Identify correlation between Excel knowledge and prior experience
- Generate insights: "Students with prior experience have 40% higher statistical confidence"

### 5.3 Phase 3: Dashboard Creation with Claude AI

**Goal:** Transform Google Sheets data into an interactive HTML dashboard

**Why HTML Dashboards?**
- No additional software required
- Shareable via web link
- Interactive visualizations using JavaScript libraries
- Professional presentation for stakeholders

**Using Claude for Dashboard Generation:**

**Step-by-Step Process:**

1. **Share Data Context with Claude**

   ```
   - Total registrations breakdown
   ```

2. **Claude Generates HTML/CSS/JavaScript**

3. **Dashboard Features**

   ```html
   <html>
   <head>
       <title>MBA Analytics Seminar Dashboard</title>
       <script src="https://cdn.jsdelivr.net/npm/chart.js"></script>

       <script>

   ```

- **Conversion Rate:** (Conversions / Total Visitors) × 100
- **Churn Rate:** (Customers Lost / Total Customers) × 100
- **CAC:** Customer Acquisition Cost
- **LTV:** Lifetime Value of a customer
- **NPS:** Net Promoter Score (customer loyalty metric)

### E. SQL Cheat Sheet for Business Analysis

```sql
-- Basic query structure
SELECT column1, column2, AVG(column3) as avg_value
FROM table_name
WHERE condition
GROUP BY column1, column2
HAVING avg_value > threshold
ORDER BY column1 DESC
LIMIT 10;

-- Common business queries
-- Top 10 customers by revenue
SELECT customer_id, SUM(purchase_amount) as total_revenue
FROM sales
GROUP BY customer_id
ORDER BY total_revenue DESC
LIMIT 10;

-- Month-over-month growth
SELECT 
    DATE_TRUNC('month', order_date) as month,
    SUM(revenue) as monthly_revenue,
    LAG(SUM(revenue)) OVER (ORDER BY DATE_TRUNC('month', order_date)) as prev_month,
    ((SUM(revenue) - LAG(SUM(revenue)) OVER (ORDER BY DATE_TRUNC('month', order_date))) 
     / LAG(SUM(revenue)) OVER (ORDER BY DATE_TRUNC('month', order_date)) * 100) as growth_pct
FROM orders
GROUP BY month;
```
